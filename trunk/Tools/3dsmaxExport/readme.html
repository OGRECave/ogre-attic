How to use the 3DS exporter

Please read 'install.txt' for information on how to install the exporter.

Features of the exporter:
  • exports meshes
  • exports bones, vertex assignements and bones animation, using the skin modifier
  • optionally exports embedded material definitions, however you are better to simply assign named materials and use an OGRE .material script to define them, since you can then use the full OGRE material properties.
  • exports everything to XML, which allows you to review or alter the exported details if you wish. The CML must be converted to OGRE binary format using XMLConverter before runtime use.
How to use it
  1. In the 'Ogre Tools' toolbar, click the 'Export to OGRE' button
  2. Click the 'Select A Mesh' button and pick the mesh you wish to export
  3. Tick the'Export Mesh' checkbox if you want to export the geometry of the object
  4. Tick the 'Export Skeleton' box if you want to export the skeleton and animations. Specify the name you want to give the animation, the range and duration.
  5. Specify an output file name which will be used as a base for all exported files
  6. Click 'Export'
This will create XML files called <outputfile>.mesh.xml and <outputfile>.skeleton.xml. The former contains the geometry, material links and bone assignments, the latter contains the definition of the skeleton and the animation which was exported. Note that it is possible to have more than one animation for the skeleton in the same skeleton file, all you have to do is export the animations to separate files, and combine the extra animations in one XML file by adding the <animation> node  and all it's children to the one <animations> node on the combined file, with a text editor.

Converting XML to binary files
Once you're happy with the files, you need to convert them to OGRE's binary formats to use them at runtime (XML is too inefficient to use for large data structures like meshes at runtime).

To do this, you need to use the XMLConverter tool, provided in the Ogre source tree under ogrenew/Tools/XMLConverter, or available as a precompiled binary if required. Simply run:
XMLConverter <xml filename>
.. on both the .mesh.xml and .skeleton.xml files, and XMLConverter will produce .mesh and .skeleton binary files out of the XML. These can then be placed into your OGRE resource locations for runtime use.

Issues
  • WARNING : Always create new bones as children of another AND don't unlink and re-link a bone. For example, if you have created a leg, copy all the bones and then use position controller in a track view to move it and so get the other leg.
  • WARNING : Don't use mirror when you create your bones because results are strange.
  • Sometimes XMLConverter reports a message about not being able to find the .skeleton file when converting a .mesh.xml. Do not worry about this, this is because the converter uses the runtime objects from OGRE itself and has not established any resource locations;
    this message will not affect the validity of the .mesh produced.
  • At every keyframe, you have to had a key for position x,y,z and rotation x,y,z, otherwise you will have an error message: "a key missing for a track" or something like that. To get around this, use the  'Add missing keys for selected object' button at the bottom of the export dialog.
  • Frame 0 must be a keyframe and at this keyframe, bones mustn't be rotated, they must be in their rest position. If you want to begin your animation with a different position, start animating at a higher frame number and tell the exporter to start from there.
  • There is sometimes a problem with mapping. Try to add a UVW map modifier.
  • Only skeletal animation is supported, so a 'skin' modifier is required.
  • A new material is available in the material editor: OGRE Material. Only this kind of material can be embedded in the .mesh; however, in any case it is better to simply name your materials and use a .material script to define the material, that way you can use all the OGRE material properties.
