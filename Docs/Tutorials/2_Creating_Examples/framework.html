The Examples Framework

The quickest way to get started with an example application using OGRE is to use the Examples framework. In the Examples\Framework folder you will find 2 files, 'ExampleApplication.h' and 'ExampleFrameListener.h'.

The ExampleApplication class has almost everything you need to get a working OGRE application running. It contains the necessary code to initialise the system, set up a render window and respond to user input for you. All you have to do is subclass it and provide an implementation for the createScene() method, in which you should set up the scene. All the examples work this way. You may also wish to override the setupResources() method to provide an alternative location for your datafiles, although since OGRE will always find resources in the current folder you may not need to do this.

Here's what a typical example application object looks like (this one comes from an old version of the EnvMap demo):

#include "..\Framework\ExampleApplication.h"

class EnvMapApplication : public ExampleApplication
{
public:
    EnvMapApplication() {}

protected:

    // Just override the mandatory create scene method
    void createScene(void)
    {
        // Create a light
        Light *l = mSceneMgr->createLight("MainLight");
        /* Accept default settings: point light, white diffuse, just set position
           NB I could attach the light to a SceneNode if I wanted it to move automatically with 
           other objects, but I don't
        */
        l->setPosition(20, 80, 50);

        Entity *ent = mSceneMgr->createEntity("knot", "knot.oof");

        // Get the material which has been loaded with the knot
        Material *mat = mSceneMgr->getMaterial("2 - Default");
        // Add a second texture layer 
        Material::TextureLayer *tl = mat->addTextureLayer("spheremap.png");
        // Blend the new layer with the one below
        tl->setColourOperation(LBO_ADD);
        // Make it an environment map
        tl->setEnvironmentMap(true);

        // Add entity to the root scene node
        mSceneMgr->getRootSceneNode()->attachEntity(ent);

    }

};

All you then need to do is to create an instance of this object in your WinMain method, and call it's 'go' method, as demonstrated in the examples.
